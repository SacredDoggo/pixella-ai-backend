Pixella AI Backend – API Documentation

Pixella AI Backend is a TypeScript, Express, and Prisma-based RESTful server for chat and user management.
Base URL: /api/v1

Authentication

Register

POST /auth/register
Register a new user.
Request Body: 
{
  "username": "string",   // 3-30 chars, only [a-zA-Z0-9_]
  "email": "string",      // valid email
  "password": "string"    // min 8 chars
}
Responses:
  • 201 Created – Success
  • 400 Bad Request – Invalid or missing fields
  • 409 Conflict – User exists

Login

POST /auth/login
Login with email or username.
Request Body: 
{
  "identifier": "string", // email or username
  "password": "string"
}
Responses:
  • 200 OK – { userId, username, email, token }
  • 400 / 404 / 500 – For errors

Authentication Header

All protected routes require
Authorization: Bearer <token>

Users

  • GET /user/getAllUsers — Auth required. Dev only.
  • GET /user — Current user info. Auth required.
  • GET /user/:user_id — User by ID. Auth required. 404 if not found.
  • PATCH /user/:user_id — Update user. Only owner. Partial body.
  • DELETE /user/:user_id — Delete user. Only owner.

Chats

  • POST /chat — Start new chat. Auth required. Optional title.
  • GET /chat/:chat_id/:limit? — Chat history. Optional limit param.
  • PATCH /chat/:chat_id — Rename chat.
  • DELETE /chat/:chat_id — Remove chat.

Messages

  • POST /message — {"text":"string"} — New chat + AI response.
  • POST /message/:chat_id — {"text":"string"} — Reply in chat.
  • POST /message/regenerate/:message_id — Regenerate AI response.
  • PATCH /message/change-prompt/:message_id — Edit/re-respond to user message.
  • DELETE /message/delete-message/:message_id — Delete message.

Status & Errors

Code Meaning
200 Success
201 Created
204 No Content
400 Bad Request
401 Unauthorized
403 Forbidden
404 Not Found
409 Conflict
500 Internal Server Error
On error: 
{ "error": "Message here" }
All time, IDs, and tokens use standards. See repo for implementation details.
